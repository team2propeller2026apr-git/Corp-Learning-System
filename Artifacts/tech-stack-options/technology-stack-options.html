Technology Stack Options

For the Corporate Learning Progress, Intervention & Compliance Tracking System, the stack should support APIs, rule evaluation, imports, dashboards, reports, audit history, RBAC, and compliance. The architecture should be modular and production-ready without overengineering.

Recommended Stack Options

Option Stack Best For Reason
Option 1: TypeScript Full Stack React + NestJS + PostgreSQL + Redis + Docker Best balanced choice for POC to production Fast development, strong typing, good API structure, and a strong ecosystem for dashboards, queues, validation, auth, and background jobs.
Option 2: Enterprise Java Angular/React + Spring Boot + PostgreSQL + Redis + Docker Enterprise-grade backend Strong for complex business rules, RBAC, audit, integrations, batch jobs, security, and long-term maintainability.
Option 3: .NET Enterprise React/Angular + ASP.NET Core + PostgreSQL/SQL Server + Redis + Docker Microsoft/Azure-heavy organizations Excellent Azure integration, strong identity support with Entra ID, good performance, mature APIs, background jobs, and enterprise tooling.
Option 4: Python Data-Friendly React + FastAPI + PostgreSQL + Redis + Celery/RQ If analytics/risk scoring may evolve toward ML Very productive for APIs, data processing, rules, reporting, and future analytics, but needs discipline for large enterprise codebases.
Option 5: Low-Code + API Backend Power Apps/Power BI + Azure Functions/API + Dataverse/PostgreSQL Fast internal enterprise prototype Useful if the main goal is quick dashboards and workflows, but less flexible for custom rules, complex integrations, and long-term product control.

Preferred Choice

Use React + NestJS + PostgreSQL + Redis + Docker, deployed on Azure Container Apps.

  • React is strong for dashboards, admin screens, profile views, and report interactions.
  • NestJS gives a structured backend with modules, guards, interceptors, validation, and clean service boundaries.
  • PostgreSQL fits employee profiles, attendance, assessments, competencies, risk results, interventions, audit logs, and reports.
  • Redis supports caching, queues, rate limiting, and background job coordination.
  • Docker keeps local, test, staging, and production deployments consistent.
  • Azure Container Apps gives production-ready managed containers without Kubernetes complexity.
